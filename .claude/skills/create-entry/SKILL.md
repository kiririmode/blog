# Create-Entry Skill

このSKILLは、読者が惹きつけられる、魅力的なブログエントリを作成するためのもの。

# Instructions

ブログエントリを作成するには、以下のステップを実行する。

1. 新規エントリの作成: 以下のコマンドで新しい下書きエントリのファイルを作成する。
   ファイルは `kiririmode.hatenablog.jp/entry/_draft/` にepoch timeを示すファイル名で作成される。

   ```bash
   npm run entry
   ```

2. ブログエントリの構造を作成する:
   上記で作成した下書きエントリファイルに対して、Plan Agentを使って具体的なエントリを記述する計画を作成する。
   構造は次のとおりとすること。
   1. **導入部**: 課題提起から入り、背景説明、解決策の概要を提示
   2. **目次**: `[:contents]` ディレクティブを挿入
   3. **本文**:
      - `#` を大きなセクションに、`##` をサブセクションに、`###` は必要な場合のみ使用
   4. **最後に**: まとめを記述する
   **テンポ・ウィットの具体的な技法は `references/voice-examples.md` を参照すること。**
   特に以下を意識する：
   - 開幕の2〜3文で読者を状況に引き込んでいるか（「〜は重要です」「〜について説明します」で始めない）
   - 長文の後に短文で締める構造が随所にあるか
   - 筆者の内なる声・自己ツッコミ・自虐ユーモアが1箇所以上あるか

3. タイトルを決める:
   タイトルガイドライン `references/title-guildeline.md` に従ってタイトルを検討し、下書きエントリのFront Matter部に記載すること。
   - タイトルは、内容が一目で理解でき、読者が思わず読みたくなる訴求力のあるものとすること。
   - 検討の際は、以下を守ること。
      - 読者が得られる価値や解決策を示す
      - 扱う技術や課題を具体的に明示する
      - 読者の「知りたい」「解決したい」という欲求を刺激する
      - 数字、具体的な成果、意外性などで訴求力を高める
   - 例
      - GoのGoroutineで並行処理を10倍高速化した話
      - terraformで見落としがちな3つのセキュリティリスク
4. 文体を鑑みて、エントリのBODYを記述する
   コンテンツガイドライン `references/contents-guideline.md` に従って記述する。

5. テンポ・ウィットチェックを行う
   書いたBODYを通読し、以下を確認する。問題があれば書き直す。
   - 「硬い」「説明的すぎる」と感じる箇所はないか
   - 同じ文末（〜です。〜です。〜です）。が3文以上続いていないか
   - 「この人面白いな」と思える瞬間が少なくとも1箇所あるか
   - ユーモアが「口調の荒さ」だけで成立していないか。長めの論理展開→短い断定や問い→不条理な締め、という落差の構造になっているか（`references/voice-examples.md` のテクニック6参照）
   - 同じテンションで最後まで書いていないか。緩急があるか

6. lintをかけ、エラーがあった場合は修正する。そのサイクルを、エラーが0になるまで繰り返す。

   ```bash
   npm run lint path/to/file.md
   ```

7. 下書きエントリファイルのFrontMatterにおけるDateフィールドを、現在時刻（JST）に置き換える
8. 変更をコミットする

   ```bash
   /commit
   ```
