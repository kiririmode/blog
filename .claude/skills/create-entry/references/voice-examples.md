# 声・テンポ・ウィットの具体例

contents-guideline.mdに書いてある原則を、具体的な例で補完する。
「なんか硬い」と感じたら、ここの例を参照すること。

## テクニック1: 長文の後に短文で締める

長い説明が続いたら、短い1文で着地させる。それだけでリズムが生まれる。

**硬い:**
> プロダクトマーケティングの目的は、ビジネス目標を達成するための戦略的なマーケティング活動を通じて、市場のパーセプションを形成し、プロダクトの定着を促進することです。

**軽い:**
> プロダクトマーケティングの目的を一言で言うと、「届ける構造を市場に実装すること」です。作ることは手段。届けることが目的。この二つが噛み合わないと、良いものが静かに埋もれていきます。

## テクニック2: 内なる声を可視化する

筆者が何を考えたか、何を感じたかを書く。「客観的な情報」だけでなく「主観的な反応」を入れる。

**人間がいない:**
> LOVEDを読むことで、プロダクトマーケティングへの理解が深まりました。

**人間がいる:**
> LOVEDを読んで、「作ることへの集中」と「届けることへの無関心」の間にあった溝が、ようやく言語化された気がしました。ずっとモヤモヤしていたやつに、名前がついた感覚。

## テクニック3: 期待外れのオチ

真面目に前置きして、最後の1文を軽くする。落差がウィットになる。

**例1 (軽い免責):**
> さらに重要な注意事項として、本文中に頻出する「お前」という二人称は、画面の前のあなた個人を指しているわけではありません。これは一般論としての架空の人物、つまり「生成AIに曖昧な指示を出しがちな人」や「生成AIを魔法の杖のように考え、何も言わなくても期待通りの成果物が出てくると思っている人」といった、我々人類の総体としての象徴的存在を指しています。したがって、この「お前」はあなたではありません。谷沢です。

**例2 (自虐):**
> 財布に優しくないですが、音楽体験の質が変わるなら、それは投資だと自分に言い聞かせています。

**例3 (雑な締め):**
> 磁性流体ドライバーという謎技術のおかげらしいです。よくわからんけど、凄い。

## テクニック4: 自己ツッコミ

自分の矛盾や失敗を先に言ってしまう。読者より先に言ってしまうことで、親近感が生まれる。

**例1:**
> 「まだ使えるのに新型買うのか」という内なる声を無視して、ポチりました。

**例2:**
> わかるわ。エンジニアとして「要件書いて」ってプロダクトマネージャーに言ってたことあるわ。プロダクトマネージャーとして、「これ作って」って言ってるわ。

## テクニック5: 開幕で「状況」を作る

「〜は重要です」「〜について説明します」は開幕殺しの典型。読者を状況へ引き込む書き出しにする。

**悪い開幕:**
> プロダクトマーケティングは現代のビジネスにおいて重要な役割を果たしています。本記事では...

**良い開幕:**
> プロダクトを作ってリリースすると、「あとはユーザが使ってくれるだろう」と思ってしまいがちです。実際、リリースした時点で満足感と達成感が得られてしまう。

**良い開幕（別パターン）:**
> 買い物って不思議なもので、買う前は「これで生活が変わる！」と期待に胸を膨らませるんですが、実際には引き出しの肥やしになることも多いわけです。

## テクニック6: 論理の丁寧さと口語の落差

これがすべての技法の根底にある原則。面白さは乱暴な口調そのものから生まれるのではなく、**丁寧な論理展開との落差**から生まれる。

構造は3段階。

1. **前提を丁寧に整理する**（長めの説明文）
2. **要所で短い断定文や問いを差し込む**（リズムを生む）
3. **少し不条理な比喩や固有名詞で落とす**（印象を残す）

落差を大きくする手法のひとつが、**大袈裟な言葉選びで意図的に格調を上げてから、mundaneなものに着地する**こと。
「我々人類の総体としての象徴的存在」という大仰な表現で読者の期待値を引き上げ、「谷沢です」で静かに着地する、あの構造。
格調の上げ幅が大きいほど、落ちたときの笑いも大きくなる。

批判の矛先は「読者個人」ではなく「曖昧な依頼」「未定義の評価語」「思考停止した期待」などの構造的な問題に向けること。口調が荒くなっても攻撃対象を間違えない。

**例（構造が見える形で）:**

> [前提の整理] でもさ、その「いい感じ」が何を意味するのか、ちゃんと言語化できていますか？
> [断定で圧をかける] 「いいコード」って何ですか？「良い設計」って何ですか？
> [不条理な締め] 「いい成果物」とかいうやつをさ、頼むから屏風から出してください。

**避けること:**

- 口調だけ荒くて中身がない文
- 同じテンションでずっと続く平板な文（読者が慣れてしまう）
- 個人攻撃・罵倒のための罵倒

---

## 避けるべきパターン

| 避けるもの | 代わりに |
| --- | --- |
| 「〜は重要です」で始まる文 | 具体的な状況や問いかけから入る |
| 「〜について説明します」 | いきなり本題に入る |
| 「まとめると、〜です」の連発 | 短い1文で締める |
| 同じ文末（〜です、〜です、〜です） | 体言止め、疑問、短文を混ぜる |
| 機能の列挙 | 使ってみた感想や反応を書く |
| 「〜と感じました」で終わるまとめ | 「だから何？」まで書く |
| 同じテンションで最後まで続ける | 丁寧→短文→落差、のリズムを作る |
