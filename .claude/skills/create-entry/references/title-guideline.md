# タイトルガイドライン

タイトルは、内容が一目で理解でき、読者が思わず読みたくなる訴求力のあるものとすること。

検討の際は、以下を守ること。

- 読者が得られる価値や解決策を示す
- 扱う技術や課題を具体的に明示する
- 読者の「知りたい」「解決したい」という欲求を刺激する
- 数字、具体的な成果、意外性などで訴求力を高める

## 例（分析型）

- GoのGoroutineで並行処理を10倍高速化した話
- terraformで見落としがちな3つのセキュリティリスク

## エッセイ型の場合

エッセイ型エントリ（`references/contents-guideline.md` の「記事のモード」参照）では、上記の例より長く・口語的なタイトルも許容される。読者への呼びかけや、記事の結論・オチをタイトルに含めてもよい。

例: 言語化が大事なこの大生成AI時代におれたち人類は言語化ができないんだから、/grill-me で言語化していこうか
