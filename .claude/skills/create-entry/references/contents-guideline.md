# コンテンツガイドライン

## 文体

- コーヒーを飲みながら友達に説明するみたいに書くこと。マーケティングっぽい言葉はなし。

このブログは、読者に価値を提供するための独自のスタイルを持つ。

- **リズムとテンポ** - 読みやすく心地よい躍動感のある文章のリズムを持つこと
- **知的なウィットとユーモア** - 言葉遊び、パラドックス、皮肉な観察、ウィットを適度に織り込むこと
- **温かみのあるトーン** - 知的でありながらも親しみやすさを持つこと
- **自然な表現** - 狙いすぎな言葉選び、偉そうな言い回しを避ける
- **平易で気取らない言葉** - 堅苦しい表現を避け、親しみやすく
- **論理的な構成** - 飛躍せず、つながりが明確な展開
- **散文優先** - AI生成感を避けるため箇条書きを最小化
- **十分な深さ** - 表面的でない、価値ある情報量

## 記事のモード

エントリは、出発点・主軸によって「エッセイ型」と「分析型」の2モードに分かれる。
構造プラン作成時（SKILLのステップ2）に、AIがどちらのモードかを提案し、ユーザーが確認・修正する。

| モード | 出発点・主軸 | 例 |
| --- | --- | --- |
| エッセイ型 | 筆者自身の体験・もやもや・気づきから始まる、自己言及的な問い（「自分(たち)はなぜ〜できないのか/してしまうのか」） | 言語化が大事なこの大生成AI時代に... |
| 分析型 | 外部の情報・データから導かれる構造分析・メカニズム解明（「Xという現象/構造はどう機能するのか」） | 理解負債、Jevonsのパラドックスを扱ったエントリ |

判断に迷う場合は、記事の冒頭1〜2段落が上記どちらの問いの形に近いかで判断する。

## 文体の基本ルール

### 分析型

- ですます調を基本とする。文章のリズムが出る場合は、である調を混ぜる。

### エッセイ型

- だ/である調＋「じゃん」「わ」「だよ」「だもん」などの口語表現をベースにする。ですます調を基本としない。
- 読者を「お前」「おれたち」「人類」のように、筆者自身も含めた総称で直接呼びかける。
- 「〜だよね？」「〜じゃん」のような問いかけ・確認のリズムを高頻度で挿入し、読者と対話するようなテンポを作る。
- 模範例は `references/essay-mode-example.md` を参照すること。

### 共通

- 事実と想像は分ける。
  - 事実:「〜となります」（確定的）、「〜です」（基本的な説明）、「〜できます」（機能や可能性）
  - 個人の見解:「僕は〜と考えています」「〜だと思います」

**コードブロック**:

````markdown
```言語名
コード
```
````

**引用**:

```markdown
> 引用文
>
> <cite>[引用元タイトル](URL)</cite>
```

埋め込みURLの記述は以下のようにします。

```markdown
[https://url:embed]
```

### 品質基準

- 文の長さ: 最大200文字を推奨
- 箇条書きを最小限にし散文で記述する
- 技術用語は初出時に定義
- コマンド出力は重要な部分を強調
- 全体としてユニークを持った文章になっていること
